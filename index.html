2 vagrant 创建私有box

1) 我们需要知道要制作box的虚拟机的真实名称, 执行：

VBoxManage list vms

“vm10-0-7-66” {1b85767d-78c9-4bbe-8090-d23c77ab94d0}
vm10-0-7-66 是我们要制作的box的虚拟机的名字

2）创建box

vagrant package —base vm10-0-7-66 —output centos_lb/7.7.box

如有box名称存在我们首先要删除box

vagrant box remove centos_lb/7.7.box

3) 建立box 并添加到本地vagrant

vagrant box add centos_lb/7.7.box centos_lb/7.7.box

==> box: Box file was not detected as metadata. Adding it directly…
==> box: Adding box ‘centos_lb/7.7.box’ (v0) for provider:
box: Unpacking necessary files from: file:///home/dev/centos_lb/7.7.box
==> box: Successfully added box ‘centos_lb/7.7.box’ (v0) for ‘virtualbox’!
[root@vm10-0-0-15 dev]# vagrant box list

已经添加进vagrant中了

删除刚生成的已经使用过box的文件

rm -fr centos_lb

4) 可能重复安装需要删除的文件夹 vm766 重复安装的虚拟机名称

rm -fr /home/dev/.vagrant/machines/vm766

3 新增挂载磁盘

1） 初始化虚拟机

停机并准备添加新硬盘

vagrant halt vm765

新增一个100g的vmdk磁盘文件

VBoxManage createhd —filename /home/dev/disk/homer66_100g —size 100000 —format VMDK

2) 创建虚拟磁盘

查看虚拟机info

VBoxManage showvminfo vm10-0-7-65

根据输出信息，大概找到 Storage Controller,查看只有ide，没有sata

增加了个 sata 驱动器

VBoxManage storagectl vm10-0-7-65 —name “SATA Controller” —add sata —portcount 5 —controller IntelAhci —bootable on

继续把硬盘插上去

VBoxManage storageattach vm10-0-7-65 —storagectl “SATA Controller” —type hdd —medium /home/dev/disk/homer65_100g.vmdk —port 1 —device 0
vm1-master 就是你虚拟机的名字，上面有提到
–storagectl 参数就是磁盘控制器的名字，参考上一条命令中的 –name
–type hdd 表示是硬盘，因为这个命令不仅仅可以插硬盘，还能插光驱等，所以要指定插的是什么
–medium 这里指定的是虚拟磁盘的文件名，想想一下你手里拿着的硬盘
–port 表示插在哪个端口，上面我们创建 sata 控制器的时候通过 –portcount 5 开放了5个端口，相当于主板上的5个sata- 接口，此时我们用第1个
–device 0 设备id，设为0 就可以了，我也不知道什么意思
如果没有错误的话，这里就成功添加了一块硬盘到虚拟机了，启动虚拟机，这里我用的是vagrant来管理虚拟机的。
vagrant up vm765
vagrant ssh vm765

剩下的就和正常服务器一样了，fdisk 格式化硬盘，然后mount到你想mout的位置

mysql 数据库初始安装

Posted on 2020-06-20 | In MYSQL

sudo mkdir -p /data/aiaas/config/mysql/
sudo bash -c ‘cat >> /data/aiaas/config/mysql/my.cnf << EOF
[mysqld]
#

Remove leading # and set to the amount of RAM for the most important data

cache in MySQL. Start at 70% of total RAM for dedicated server, else 10%.

innodb_buffer_pool_size = 128M

#

Remove leading # to turn on a very important data integrity option: logging

changes to the binary log between backups.

log_bin

#

Remove leading # to set options mainly useful for reporting servers.

The server defaults are faster for transactions and fast SELECTs.

Adjust sizes as needed, experiment to find the optimal values.

join_buffer_size = 128M

sort_buffer_size = 2M

read_rnd_buffer_size = 2M

skip-host-cache
skip-name-resolve
datadir=/var/lib/mysql
socket=/var/lib/mysql/mysql.sock
secure-file-priv=/var/lib/mysql-files
user=mysql
max_allowed_packet=100M

Disabling symbolic-links is recommended to prevent assorted security risks

symbolic-links=0

log-error=/var/log/mysqld.log
pid-file=/var/run/mysqld/mysqld.pid
sql_mode=ONLY_FULL_GROUP_BY,STRICT_TRANS_TABLES,ERROR_FOR_DIVISION_BY_ZERO,NO_AUTO_CREATE_USER,NO_ENGINE_SUBSTITUTION
EOF’

sudo bash -c ‘cat >> /data/aiaas/shells/docker-compose-mysql.yml << EOF
version: ‘3’

services:

mysql:
image: hub.lingban.cn/mysql:5.7
restart: always
privileged: true
volumes:

   - /data/aiaas/data/mysql/mysql_init:/docker-entrypoint-initdb.d
   - /data/aiaas/data/mysql/data:/var/lib/mysql
   - /data/aiaas/config/mysql/my.cnf:/etc/my.cnf
   - /data/aiaas/logs/mysql/log:/var/log
environment:
   TZ: Asia/Shanghai
   MYSQL_ROOT_PASSWORD: Lingban2018_AIAAS
ports:
  - "3306:3306"
logging:
  options:
    max-size: '100m'
    max-file: '10'

volumes:
app_net:
external: true
EOF’

cd /data/aiaas/shells/
sudo docker-compose -f docker-compose-mysql.yml up -d
